# Scenario 1: Cross-Border Inference Leakage

## Overview

An enterprise deploys a generative AI assistant to help analysts summarize internal reports. The assistant uses an external LLM API hosted by a global provider.

Sensitive financial data is included in prompts submitted to the model.

The AI provider processes inference requests in multiple geographic regions.

## Threat Path

1. User submits sensitive enterprise data to the AI assistant
2. Prompt is forwarded to external LLM provider
3. Request is routed to an inference cluster in another jurisdiction
4. Prompt data is logged or stored for model improvement

## Risks

- Data residency violations
- Unauthorized international data transfer
- Exposure to foreign legal frameworks
- Potential retention of confidential data by provider

## Business Impact

- Regulatory penalties
- Loss of sensitive financial information
- Contractual breaches with customers or partners

## Mitigation Strategies

- Enforce regional inference boundaries
- Use private model hosting when possible
- Implement prompt filtering for sensitive data
- Require provider contractual guarantees on data retention

## Governance Alignment

Relevant controls include:

- NIST AI RMF: Govern and Manage functions
- Data classification policies
- Vendor risk management processes
