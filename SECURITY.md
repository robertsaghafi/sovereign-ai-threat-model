# Security Policy

## Purpose

The purpose of this repository is to provide a threat modeling framework for sovereign AI and agentic AI systems. While the project primarily contains documentation, frameworks, and governance artifacts, it is important to maintain responsible security practices for any code, templates, or tools included in this repository.

This document outlines how security vulnerabilities should be reported and handled.

---

# Supported Versions

This repository is under active development. The most recent version available in the main branch should always be considered the supported version.

Security issues discovered in previous commits should still be reported if they may impact users of the framework.

---

# Reporting a Vulnerability

If you discover a security vulnerability related to this repository, please report it responsibly.

Examples include:

- Vulnerabilities in any code or scripts included in the repository
- Security weaknesses in example implementations
- Issues that may expose sensitive information
- Design flaws that could introduce security risk in downstream implementations

Please **do not open public issues for security vulnerabilities**.

Instead, report the issue privately by contacting:

**Email:** robert.saghafi@aerisintelligentsystems.ai 

When reporting a vulnerability, please include the following information:

- Description of the vulnerability
- Steps required to reproduce the issue
- Potential impact of the vulnerability
- Suggested mitigation if available

Providing clear and reproducible information will help us investigate and resolve issues more efficiently.

---

# Responsible Disclosure

We request that researchers and contributors follow responsible disclosure practices.

Please allow time for the issue to be reviewed and addressed before publicly disclosing details of the vulnerability.

Our process generally follows these steps:

1. Acknowledge receipt of the vulnerability report
2. Assess the severity and impact
3. Develop a mitigation or fix
4. Publish updates or documentation as needed

---

# Scope

This security policy applies to:

- Source code contained in this repository
- Example scripts and templates
- Documentation artifacts that may influence security implementations

This repository provides **guidance and frameworks**, not production software. Organizations adopting the framework remain responsible for validating security implementations within their own environments.

---

# Security Best Practices

Users implementing concepts from this repository should follow established security practices, including:

- Performing independent security reviews of implementations
- Validating controls against relevant governance frameworks
- Monitoring AI system behavior in production environments
- Maintaining secure configuration and access controls

Security architectures should always be adapted to the specific risk profile of the organization.

# Acknowledgements

We appreciate responsible security research and contributions that help improve the safety and governance of AI systems.

Responsible reporting helps strengthen the broader ecosystem for secure and sovereign AI adoption.
